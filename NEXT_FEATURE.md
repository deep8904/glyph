# Next Feature — V2 Feature 2

**Project Pages & Devlogs.** Now that every developer has an identity (`/dev/[username]`) and an optional primary project, the highest-leverage next build is full **project pages with append-only devlogs** — the build-in-public loop that itch.io proved works and Indie Hackers turned into a retention engine. Concretely: a public project page at `/dev/[username]/[project-slug]` showing cover, description, engine/genre/stage, and a reverse-chronological devlog feed; an authenticated composer to post structured devlog updates (title, body/markdown, optional image) appended to the project; and surfacing the latest devlog on the developer's profile and (later) a global community feed. This deepens single-user value immediately (a dev can document progress on day one), feeds the future Community Feed and Discovery features, and reuses the existing `projects` table (add a `devlogs` table referencing `project_id`). Build it before collaboration/events because devlogs are what make a profile *worth following* — they convert a static résumé into a living presence that an audience returns to.
